Parking Space Detection Workflow

This project turns georeferenced aerial imagery and ArcGIS parking-space labels into a trained YOLO11s-OBB model, review crops, attention maps, and map-linked outputs. The notebooks automate the steps; the goal of this page is to make the logic easy to follow.

Folder note: unzip or upload the provided ParkingSpaceDetection folder to Google Drive; the notebooks use that folder name directly.

Big Picture

The workflow has three main ideas. First, a large region of interest is split into smaller georeferenced tiles. Second, global parking-space polygons are converted into YOLO oriented bounding-box labels for each tile. Third, the trained detector produces predictions that are converted into human-friendly review material and georeferenced outputs.

Workflow Steps

1

Split the ROI GeoTIFF into smaller tiles

The original ROI.tif is too large to use directly for training. It is split into many smaller tile_*.tif files. Each tile keeps georeferencing information, which means pixel positions can still be converted back to real-world coordinates later.

InputROI.tif
Processtile into fixed-size GeoTIFF images
Outputtile_001.tif, tile_002.tif, ...
georeferenced input tile GeoTIFFs
2

Label parking spots in ArcGIS Pro

Parking spaces are drawn once as global polygons in ArcGIS Pro. The result is exported as a GeoJSON file such as parking_spots.geojson. These are the ground-truth annotations used for model training.

Inputtile imagery visible in GIS
Processmanually draw parking-space polygons
Outputparking_spots.geojson
human labels ArcGIS Pro GeoJSON polygons
3

Convert global annotations into YOLO OBB labels

The global GeoJSON polygons are matched to each tile. For every parking-space polygon that belongs in a tile, the scripts convert real-world coordinates into normalized image coordinates. This creates YOLO oriented bounding-box labels, where each label stores the four rotated corners of a parking space.

Inputtile_001.tif + parking_spots.geojson
Processclip polygons to tiles and normalize corner coordinates
Outputtile_001.png + tile_001.txt
Notebook 1 01_tif_to_png.py 02_geojson_to_yolo_obb.py 03_make_train_val_split.py
4

Train YOLO11s-OBB

YOLO11s-OBB learns to detect rotated parking spaces from the image-label pairs. The model is trained on PNG tile images and YOLO OBB text labels. The important saved result is best.pt, which is the fine-tuned model checkpoint used for prediction.

Inputdataset/images + dataset/labels
Processtrain YOLO11s-OBB on train split
Outputruns/obb/train/weights/best.pt
Notebook 1 04_train_obb.py fine-tuned model
5

Validate detection performance

The dataset is split into training and validation sets, usually 80/20. Training images teach the model. Validation images are held out so we can check whether the model detects parking spaces on images it did not train on.

Inputbest.pt + validation images
Processpredict and validate against held-out labels
Outputprediction labels, preview images, validation metrics
Notebook 1 05_predict_obb.py 06_predictions_to_geojson.py 07_dedupe_predictions.py
6

Extract detected areas as upright PNG crops

Raw OBB predictions are rotated boxes on large tiles. For review, each detected parking space is cut out, rotated upright, placed on a consistent canvas, and sorted by confidence. This creates a gallery that is much easier to inspect than full-tile predictions.

Inputprediction .txt labels + source images
Processwarp each rotated OBB into an upright crop
Outputreview_crops/all + manifest.csv
Notebook 2 confidence gallery review manifest
7

Generate attention maps for explainability

Attention maps show which image regions influenced the model for each detection. The crop-level attention view is useful for quick inspection, while full-tile heatmaps preserve the surrounding context so the detection can still be understood in its original location.

Inputbest.pt, validation images, manifest.csv
Processgenerate per-detection CAM overlays
Outputreview_crops_cam/all and full-tile heatmaps
Notebook 3 13_generate_obb_cam_crops.py attention maps
8

Create a georeferenced review website

The final visualization combines model outputs with georeferencing. Detections can be linked back to real-world coordinates, crop images, attention maps, and Google Maps links. This makes the results easier to communicate to people who do not want to read model files or notebooks.

Inputmanifest files, crops, CAM images, GeoTIFF transforms
Processassemble static HTML review page
Outputwebsite with mapped detections and review links
12_build_review_site.py georeferenced website Google Maps links
